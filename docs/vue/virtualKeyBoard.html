    Vue全局组件引入

Vue.component('keyboard', keyboard)

    # 组件使用

    该组件目前只是简单的实现，操作使用流程大致可以分为以下几个步骤：

    1. 在父组件中引入keyboard组件
    2. 绑定触发调用虚拟键盘的元素id(triggerId)
    3. 添加用来监听键盘值的方法watchFinalValue(value)

    ​ 虚拟键盘是提供了一种承载输入的工具，他并不能解决你具体的业务，也就是说，虚拟键盘并不提供像el-input一样提供type 属性来解决你想要的"颜色"，具体的业务交给了用户自己去自由发挥。你要是使用虚拟键盘的话，几乎必须要添加监听的方法来获取虚拟键盘从开始到结束这段时间所触发的值，来满足一个完整的键盘所必备的功能。

    # Attributes

    # triggerId

    ​ 该属性主要是在页面上绑定需要触发虚拟键盘显示的元素id，如果页面中需要多处使用虚拟键盘，那么这里可以写多个触发id，用英文逗号进行分割开。

    ​ 需要注意的是，这里的填写的id顺序其实是有一定约束含义的，具体会配合后面的方法setTriggerId一起使用，具体用法放在后面的方法中概述。

    # keyId

    ​ 这个属性其实就是指定虚拟键盘这个DIV的id，这个属性目前来看没什么用，但是在后续的更新优化中很可能会用到。

    # Methods

    # watchFinalValue(value)

    ​ 这个方法是在父组件中引用时需要监听子组件值变动的方法，也就是在“点击”虚拟键盘各个键位的时候，会返回一个当前虚拟键盘打开到结束这一段时间内，所捕获到的值。

    # setTriggerId(id)

    ​ 这个方法返回了一个id,这个id是在触发虚拟键盘时，第几个keyboardid，这个方法是配合triggerId属性使用的，返回的值是triggerId中的索引下标加1。在这个属性中的顺序，是返回这个id的关键，这也是在同一个页面中引入多个虚拟键盘的关键，前面已经提到，虚拟键盘只是提供了一种输入的工具，如果用户想要多次引用组件怎么办？多次引入？这显然是不合理的，参考windows的虚拟键盘就能理解，这个组件一定是singleton。

    # 示例

    <template>
  ......
  <!-- keyboard -->
  <keyboard 
            triggerId="test1,password" 
            keyId="KeyBoard" 
            v-on:watchFinalValue="watchFinalValue"
            v-on:setTriggerId="setTriggerId"
            >
    </keyboard>
    <el-input
              id = "test1"
              name="test1"
              v-model="this.loginForm.account"
              placeholder="请在此输入账号名称">
      </el-input>
      <el-input
                name="password"
                id="password"
                v-model="loginForm.password"
                autocomplete="on"
                placeholder="请在此输入账号密码">
    	</el-input>

  ......
</template>
<script>
  export default {
    methods: {
      setTriggerId(id){
        this.whichTrigger = id;
      },
      watchFinalValue(value){
        switch(this.whichTrigger){
          case 0:
            console.log("there is no trigger for keyboard!!")
            break;
          case 1:
            console.log("this is the first trigger for keyboard! value:"+value);
            this.loginForm.account=value
            break;
          case 2:
            console.log("this is the second trigger for keyboard! value:"+value)
            this.loginForm.password=value
            break;
          default:
        }
      }
  }
</script>
