    va_list函数及其用法

    Ryan 3/16/2022

    在阅读XRDP源码的过程中我发现里面大量使用了g_snprintf这个方法，搞不明白这个是干什么用的，于是翻箱倒柜终于给他搞明白了，话不多说上代码

    • va_list
    • vastart()、vaarg（）、va_end()
    • vsnprintf

    int
g_snprintf(char *dest, int len, const char *format, ...)
{
    int err;
    va_list ap;

    va_start(ap, format);
    err = vsnprintf(dest, len, format, ap);
    va_end(ap);

    return err;
}

    在这个方法里面，最主要的就是va_list,va_start(),va_end 这三个东东，我们一个一个介绍：

    # va_list

    #ifndef _VA_LIST
typedef __builtin_va_list va_list;
#define _VA_LIST
#endif

    2

    4

    在MacOS下这个定义是这样的，他就是一个__builtin_va_list 宏，这个东西很复杂，加builtin前缀的都是编译器内置函数。其实际就是利用函数参数入栈的顺序来计算参数地址，从而获知参数值，具体的实现在不同的cpu架构下是不一样的，函数参数是以数据结构:栈的形式存取,从右至左入栈。首先是参数的内存存放格式：参数存放在内存的堆栈段中，在执行函数的时候，从最后一个开始入栈。因此栈底高地址，栈顶低地址，举个例子如下：void func(int x, float y, char z);那么，调用函数的时候，实参 char z 先进栈，然后是 float y，最后是 int x，因此在内存中变量的存放次序是 x->y->z，因此，从理论上说，我们只要探测到任意一个变量的地址，并且知道其他变量的类型，通过指针移位运算，则总可以顺藤摸瓜找到其他的输入变量。

    这只是X86架构下的编译器，并不是所有机器都是这样的。打个比方，在32位arm系统中，ABI规定函数参数不超过4个的情况下，保存在寄存器中，所以，靠C语言是不行的（不能直接访问寄存器）,多出来的进入栈空间，这一点和x86的规定完全不一样，所以不同系统的变长参数的实现必然不一样。这里我们只需要知道这个函数是用来获取参数地址的即可。

    # va_start()、va_arg（）、va_end()

    这三个函数为什么要放一起呢？因为这三个函数其实是配合起来使用的，

    void va_start(va_list ap, last_arg) 初始化 ap 变量、last_arg 是最后一个传递给函数的已知的固定参数，即省略号之前的参数。这个宏必须在使用 va_arg 和 va_end 之前被调用。举个栗子：

    #include<stdarg.h>
#include<stdio.h>

int sum(int, ...);

int main(void)
{
   printf("10、20 和 30 的和 = %d\n",  sum(3, 10, 20, 30) );
   printf("4、20、25 和 30 的和 = %d\n",  sum(4, 4, 20, 25, 30) );

   return 0;
}

int sum(int num_args, ...)
{
   int val = 0;
   va_list ap;
   int i;

   va_start(ap, num_args);
   for(i = 0; i < num_args; i++)
   {
      val += va_arg(ap, int);
   }
   va_end(ap);
 
   return val;
}

    num_args是一个变长参数，va_start(ap)初始化了ap变量后，使用va_arg(ap,int)来接收这个参数，第二个参数int表示参数类型，最后调用va_end(ap)。如果这里不掉用的va_end的话则结果为未定义。所以小伙伴们一定要配合这几个函数。然后我们再看vsnprintf()这个方法。

    # vsnprintf

    在上述代码中，这个函数一共有四个入参。第一个参数buffer表示输出写入的位置，第二个参数len表示写入的长度，第三个参数format表示格式规范，第四个参数ap表示指向参数列表的指针，其实这个函数还有个重载函数有第五个参数，表示指定区域格式。

    顺带说一下这个方法适用于窄字符，宽字符使用_vsnwprintf()。
