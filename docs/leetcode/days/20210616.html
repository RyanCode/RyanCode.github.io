    力扣打卡2021-6-16

    Ryan 6/16/2021 每日一题

    # 题目

    See More

    亚历克斯和李用几堆石子在做游戏。偶数堆石子排成一行，每堆都有正整数颗石子 piles[i] 。

    游戏以谁手中的石子最多来决出胜负。石子的总数是奇数，所以没有平局。

    亚历克斯和李轮流进行，亚历克斯先开始。 每回合，玩家从行的开始或结束处取走整堆石头。 这种情况一直持续到没有更多的石子堆为止，此时手中石子最多的玩家获胜。

    假设亚历克斯和李都发挥出最佳水平，当亚历克斯赢得比赛时返回 true ，当李赢得比赛时返回 false 。

    来源：力扣（LeetCode） (opens new window)

    • 题目
    • 分析
      • 方法一：动态规划
      • 方法二：博弈论

    示例

    输入：[5,3,4,5]
输出：true
解释：
亚历克斯先开始，只能拿前 5 颗或后 5 颗石子 。
假设他取了前 5 颗，这一行就变成了 [3,4,5] 。
如果李拿走前 3 颗，那么剩下的是 [4,5]，亚历克斯拿走后 5 颗赢得 10 分。
如果李拿走后 5 颗，那么剩下的是 [3,4]，亚历克斯拿走后 4 颗赢得 9 分。
这表明，取前 5 颗石子对亚历克斯来说是一个胜利的举动，所以我们返回 true 。

    # 分析

    # 方法一：动态规划

    动态规划要求上一个状态和下一个状态之间是连续的，首先我们需要找到初始状态和状态转移方程。

    并且这道题目和预测赢家 (opens new window)很相似，我们可以借助这个题目的解题方式，我们假设dp[i] [j]代表第i堆石子到第j堆石子，两个玩家之间石子数量差值的最大值。

    注意

    当i<=j时，dp[i] [j]才有意义，并且 i=j 时，两个玩家插值为piles[i]=piles[j]。即dp[i] [i]=piles[i];

    当i<j时，dp[i] [j]=max(piles[i]-dp[i+1] [j],piles[i]-dp[j] [j-1]);

    class Solution {
    public boolean stoneGame(int[] piles) {
        int length = piles.length;
        int[][] dp = new int[length][length];
        for (int i = 0; i < length; i++) {
            dp[i][i] = piles[i];
        }
        for (int i = length - 2; i >= 0; i--) {
            for (int j = i + 1; j < length; j++) {
                dp[i][j] = Math.max(piles[i] - dp[i + 1][j], piles[j] - dp[i][j - 1]);
            }
        }
        return dp[0][length - 1] > 0;
    }
}

    # 方法二：博弈论

    我们来思考一下，因为数组得长度是偶数，石子得数量又是奇数，那么必定有一个人会胜出这场游戏。题目中还有一个很重要的限制条件：只能从数组首端或者末端取元素。我们把数组中的数分为奇数组和偶数组，如果亚历克斯第一次取piles[0]（奇数组，第一个元素），那么下一次李只能取第二个元素或者第piles.length个元素，也就是偶数组。那么亚历克斯下一次完全可以取另一个奇数组元素，后面一直取奇数组元素。同理，亚历克斯如果第一次取piles[piles.length-1]（偶数组），那么整局游戏也可以完全都取偶数组元素，这样一来，结果完全是可预测的：第一次取元素的人一定会胜出。因为我们完全可以在一开始就计算出来奇数组会赢还是偶数组，接下来只要一直取会胜出的一组数据即可。

    public boolean stoneGame(int[] piles) {
        return true;
